# 割草机通讯协议-MQTT

## 1.概述

### 1.1.协议说明

​	本协议用于上位机跟下位机数据交互，上位机可根据协议获取数据和下发指令。

### 1.2.通讯技术

​	采用MQTT协议，以json 字符串数据格式进行数据传输,以主题订阅的形式进行数据交互

### **2.2.服务器发布命令主题**

**主题名称：RobotCommand/robotId（服务器发布）**

割草机指令一次性只能下发一个指令，必须等待回复后,才能下发下一条指令

|    字段名称 | 字段描述                                        |
| ----------: | ----------------------------------------------- |
|     robotId | 机器Id                                          |
| commandCode | 指令编号：服务器中的唯一code                    |
|     command | 指令内容：对应蓝牙指令协议中的command的指令内容 |

**数据请求示例：**

```c
{
  "commandCode": "1729141982671669237",
  "robotId": "C0E8D2226980",
  "command": "$SYSTEM"
}
```

# 割草机通讯协议-蓝牙

## 1.概述

### 1.1.协议说明

​	本协议主要用于割草机与手机APP进行蓝牙通讯，同时与上位机进行MQTT通讯时也可采用本协议的指令内容进行指令发送。

### 1.2.注意事项

​	利用本协议发送指令时，请注意如下规范：

- [ ] **请求指令格式：**{"command":"**请求指令内容**"}\r\n\

- [ ] **响应指令格式：**{"command":"**请求指令内容**","data":"**响应数据**","code":"**响应状态码**","message":"**状态码描述**"}\r\n

  请求指令内容：代表原请求指令，割草机响应时会原封不动响应回去。

  响应数据描述：当机器做读取查询操作时，才会有响应数据。做修改、控制等操作时响应数据为空。

  响应状态码：状态码为“00”时，代表请求成功。其他状态码都为失败。

  状态码描述：每个响应状态码的文字描述
